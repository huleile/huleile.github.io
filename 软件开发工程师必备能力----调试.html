软件开发工程师必备能力----调试

2015-12-02 技巧 快捷键, 技巧 0 评论 字数统计: 292(字) 阅读时长: 1(分)

一个不会调试的程序员不是好程序员，这个我信！
作为一个初级开发工程师，我之前的调试方式都是alert? console.log? debug? system.println?
这样确实不是太方便，而且对问题的追踪能力真的是很弱，很费时间和精力，影响开发效率。

学会打断点进入调试模式，可以让CPU降下速度，将内存爆露给开发者，开发者从而可以一步一步的查看内存信息，进而追踪问题，这样即能快速的定位到我们想要的位置也可以准确的找到问题所在，以后要多加努力，熟练使用前后端调试方法，让自己成为一个优秀的程序员。

常配置的快捷键：
F1 : 打断点/取消断点 Toggle breakpoint
F2/F3: 跳到前/后一个编辑的位置（当前文件/其他文件） Navigate forword/back
F5: 跳到下一个断点处执行 run to cursor
F6: 单步执行 step over
F7: 进入函数内部执行 step into
F8: 跳出函数执行 step out
F9: debug 调试
F10: 运行 run
F11: 重新运行 rerun
